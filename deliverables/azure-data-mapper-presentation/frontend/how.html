How

From schema to XSLT

Data Mapper is a visual authoring layer over XSLT. The workflow still needs a map artifact at runtime, so the generated files and folders are part of the design.

The artifact flow

Source schema Target schema .lml design .xslt transform Workflow action

What the mapper saves

When you save a map, Visual Studio Code creates a Data Mapper definition and a generated XSLT file. Review both in pull requests because they answer different questions.

ArtifactPurpose
Artifacts/MapDefinitions/*.lmlVisual source used by the mapper canvas.
Artifacts/Maps/*.xsltExecutable transform used by the workflow runtime.
Artifacts/Schemas/*XML or JSON schemas used for source and target trees.
Artifacts/DataMapper/Extensions/*Inline XSLT and custom function extensions.

How a workflow calls the map

For Data Mapper-created maps, use the Data Mapper Operations action named Transform using Data Mapper XSLT. Microsoft calls out this distinction because maps created by other tools use the XML Operations transform path.

Keep Map Source as LogicApp and choose the generated .xslt map name from the workflow action.
