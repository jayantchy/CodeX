Sample

A complete order mapping

This example walks through a full mapping scenario: source and target schemas (XSD & JSON), the .lml map definition, and the generated XSLT — everything you commit to Git.

Source payload

{
  "partnerOrder": {
    "id": "PO-1042",
    "orderedOn": "20260608",
    "buyer": { "name": "Contoso Retail" },
    "items": [
      { "sku": "A-100", "qty": 2, "price": 19.5 },
      { "sku": "B-200", "qty": 1, "price": 45.0 }
    ]
  }
}

Target payload

{
  "orderId": "PO-1042",
  "orderDate": "2026-06-08",
  "customerName": "Contoso Retail",
  "lines": [
    { "itemCode": "A-100", "quantity": 2, "lineTotal": 39.00 },
    { "itemCode": "B-200", "quantity": 1, "lineTotal": 45.00 }
  ],
  "grandTotal": 84.00
}

Map definition — the .lml file

The .lml file is the YAML-based design source that the Data Mapper visual editor reads and writes. It tracks schema references, connections, and function nodes. This is what you commit as the "authoring source" — the generated .xslt is the "runtime artifact".

File location: Artifacts/MapDefinitions/OrderMapping.lml 
# Azure Logic Apps Data Mapper — Map Definition
# This YAML is managed by the visual designer.
# Manual edits are possible but should be verified by reopening the map.

$version: 1
$input: Artifacts/Schemas/src_PartnerOrder.json
$output: Artifacts/Schemas/tgt_CanonicalOrder.json
$sourceSchema: src_PartnerOrder
$targetSchema: tgt_CanonicalOrder

# Direct mappings
/root/partnerOrder/id = /root/orderId

# Function: date normalization via custom function
/root/partnerOrder/orderedOn = dm:iso-date(/root/partnerOrder/orderedOn)
dm:iso-date(/root/partnerOrder/orderedOn) = /root/orderDate

# Nested property mapping
/root/partnerOrder/buyer/name = /root/customerName

# Repeating element mapping (auto-creates a loop)
/root/partnerOrder/items/* = /root/lines/*

# Child item mappings within the loop
/root/partnerOrder/items/*/sku = /root/lines/*/itemCode
/root/partnerOrder/items/*/qty = /root/lines/*/quantity

# Calculated field: qty × price
dm:line-total(/root/partnerOrder/items/*/qty, /root/partnerOrder/items/*/price) = /root/lines/*/lineTotal

# Aggregation: sum of all line totals
sum(/root/lines/*/lineTotal) = /root/grandTotal
Important: The .lml format is managed by the Data Mapper extension. The exact syntax may evolve between extension versions. Treat this as a reference representation — the canonical format is whatever the current extension generates.

Generated XSLT — the runtime artifact

When you save the map, Data Mapper compiles the .lml into this XSLT 3.0 file. This is what the workflow actually executes at runtime.

File location: Artifacts/Maps/OrderMapping.xslt 
<?xml version="1.0" encoding="utf-8"?>
<xsl:stylesheet version="3.0"
  xmlns:xsl="http://www.w3.org/1999/XSL/Transform"
  xmlns:xs="http://www.w3.org/2001/XMLSchema"
  xmlns:dm="http://schemas.microsoft.com/azure/logic-apps/data-mapper">

  <xsl:output method="xml" indent="yes" />

  <!-- Custom function: ISO date formatting -->
  <xsl:function name="dm:iso-date" as="xs:string">
    <xsl:param name="dateText" as="xs:string" />
    <xsl:sequence select="
      concat(
        substring($dateText, 1, 4), '-',
        substring($dateText, 5, 2), '-',
        substring($dateText, 7, 2)
      )
    " />
  </xsl:function>

  <!-- Custom function: line total calculation -->
  <xsl:function name="dm:line-total" as="xs:decimal">
    <xsl:param name="qty" as="xs:decimal" />
    <xsl:param name="price" as="xs:decimal" />
    <xsl:sequence select="round($qty * $price * 100) div 100" />
  </xsl:function>

  <xsl:template match="/">
    <CanonicalOrder>
      <OrderId>
        <xsl:value-of select="/PartnerOrder/Id" />
      </OrderId>
      <OrderDate>
        <xsl:value-of select="dm:iso-date(/PartnerOrder/OrderedOn)" />
      </OrderDate>
      <CustomerName>
        <xsl:value-of select="/PartnerOrder/Buyer/Name" />
      </CustomerName>
      <Lines>
        <xsl:for-each select="/PartnerOrder/Items/Item">
          <Line>
            <ItemCode><xsl:value-of select="SKU" /></ItemCode>
            <Quantity><xsl:value-of select="Qty" /></Quantity>
            <LineTotal>
              <xsl:value-of select="dm:line-total(Qty, Price)" />
            </LineTotal>
          </Line>
        </xsl:for-each>
      </Lines>
      <GrandTotal>
        <xsl:value-of select="
          sum(for $item in /PartnerOrder/Items/Item
            return dm:line-total($item/Qty, $item/Price))" />
      </GrandTotal>
    </CanonicalOrder>
  </xsl:template>

</xsl:stylesheet>

Mapping notes

Direct copy

id becomes orderId, and buyer.name becomes customerName. Simple one-to-one mappings.

Date normalization

Uses the custom dm:iso-date() function to format YYYYMMDD into ISO-style YYYY-MM-DD before downstream systems consume it.

Repeating lines

Map parent arrays explicitly. The loop is visible in the .lml and compiles to xsl:for-each in the generated XSLT.

Calculated fields

dm:line-total(qty, price) computes each line amount with rounding. grandTotal is a sum aggregation across all lines.
